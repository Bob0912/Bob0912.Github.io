Java：让我们来一起面向对象

找到人们真正关心什么，而非口头说关心什么。

——《魔鬼经济学合集》

昨天我们简单的说了 Java基础语法的一个大致入门，接下来我们会讲到Java的特性、异常处理、爬虫 等等。

我们在学习计算机语言时，时不时会说到的一个特性那就是面向对象，如C语言不是面向对象的，C++和Java都是面向对象的，那你还记得什么是面向对象吗？

目录：

  1. 什么是面向对象？

  2. 面向对象和面向过程的区别？

  3. Java中的对象和类

一.什么是面向对象？ ** 面向对象在实际开发过程中可以看作为一种软件开发方法，或者是一种编程范式。 **

面向对象的概念和应用已超越了程序设计和软件开发，扩展到其他领域如数据库系统、应用结构、应用平台、分布式系统、网络管理结构、CAD技术、人工智能等。面向对象实际上也是计算机编程技术发展到一定阶段后的产物。

小编将面向对象的理解分为两个点：

1.构成问题事务并 分解成各个对象 。

2.描述某个事务在整个 **解决问题的步骤中的行为。在实际生活中，我们将对象看作是某个具体的人或事物，但在我们的程序设计中，对于 面向对象的设计通常包括了对象的属性和动作 **，也可以称之为属性和动作的结合体。二.面向对象和面向过程的区别？

前面讲到面向对象是 计算机编程技术发展到一定阶段后的产物，那么与之对应的便是面向过程，那么面向过程又是啥呢？

面向过程讲述的是 一种以过程为中心的编程思想。 比如某些事物的发生都是以什么正在发生为主要目标进行的编程，不同于面向对象的是谁在受影响。

那么面向过程小编总结为三个小点：

1.面向过程编程需要我们 分析出解决问题的步骤。

2.然后使用函数把这些步骤 一步步实现。3.编程的主要重心放在完成的每个过程上。

区别于面向对象，主要的区别有：

1. 面向过程就是步骤 ，就是解决问题的按部就班。

2. 面向对象 关注的是解决问题所需要的对象。

3.面向过程就是自己办事，面向对象就是托人办事。

To：面向对象和面向过程还有一个明显区别便是封装、继承和类。

三.Java中的对象和类说到Java中的对象，就必然会说到Java中的类，而想要了解Java中的类，也得先了解实例变量，那么什么是实例变量呢？ ** 我们通常将实例变量声明在一个类中，但在方法、构造方法和语句块之外，且无static修饰。 **

在实例变量中，我们将数值型变量默认值为0；布尔型默认为false；引用类型默认值为null。

实例变量属于该类的对象，必须产生该类对象，才能调用实例变量。

To：字符串属于引用类型。

代码块实例:

package self2;public class Student {  int age;  String name;  boolean flag;
  public static void main(String[] args) {    Student s = new Student();    System.out.println("年龄="+s.age);    System.out.println("name="+s.name);    System.out.println("flag="+s.flag);  }}

运行结果：

我们在上面提到过 静态变量是独立于方法之外的变量，用static修饰，静态变量，也可以叫做 类变量。 ** 但是static不能修饰局部变量，比如教室中小明自己的书就是实例变量，教室中的投影仪就是静态变量。 **
